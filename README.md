# securetext.io

End-to-end encrypted text sharing between a sender and receiver, through a lightweight middleman server.

## How it works

1. A **sender** opens the app and gets a shareable link
2. The **receiver** opens the link, generates an RSA-2048 key pair, and sends the public key to the sender
3. The **sender** generates a 128-bit AES-GCM key, encrypts the message, then encrypts the AES key with the receiver's RSA public key
4. Both the encrypted message and RSA-encrypted AES key are transmitted to the receiver
5. The **receiver** decrypts the AES key with their private RSA key, then decrypts the message

The server only relays encrypted data — it never sees the plaintext. As an added security measure, 3 random "magic numbers" are generated by the receiver for the sender to verify before sending.

## Tech stack

- **Server:** Node.js, Fastify, WebSocket (`ws`)
- **Client:** Vanilla JS, Alpine.js, Web Crypto API
- **Language:** TypeScript (server)

## Development

```bash
npm install
npm run dev       # start dev server with hot reload
npm test          # run tests
npm run typecheck # type-check without emitting
npm run build     # compile TypeScript to dist/
npm start         # run compiled server
```

Requires Node.js 22+.

## Docker

```bash
docker build -t securetext .
docker run -p 3000:3000 securetext
```
